# Principles Index

The eight Knuth-style engineering principles, in the order they appear in the [[Engineering Philosophy MOC]].

## Reasoning principles (how we think about correctness)

- [[Human Understanding First]] — narratable design, exposed invariants
- [[Reason About Correctness]] — argue for it; don't hope for it
- [[Explicit Not Magical]] — clear control flow, debuggable inspection
- [[Reasoning Beside Code]] — produce the surrounding argument

## Testing principles (how we attack the program)

- [[Normal Usage Is Not Testing]] — typical use only walks the obvious paths
- [[Tests Should Make Programs Fail]] — adversarial posture finds important bugs
- [[Sharp Oracles]] — exact, decisive correctness criteria
- [[Regression Discipline]] — every bug strengthens the suite, permanently
